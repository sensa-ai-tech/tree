# VetKnowledgeTree 專案模型實測計畫

> 版本：v1.0 | 日期：2026-02-15 | 對象：獸醫師團隊 5-10 人

---

## 一、實測目標

1. **知識模型驗證**：確認六層知識架構（Layer 0-5）是否符合獸醫臨床教學邏輯
2. **內容品質評估**：AI 生成的教學內容在醫學正確性、深度、台灣在地化的表現
3. **互動體驗回饋**：測驗引擎、病例闖關、間隔重複等互動功能的使用體驗
4. **遊戲化設計驗證**：XP 系統、成就系統是否有效激勵學習動機

---

## 二、實測時程

| 階段 | 時間 | 內容 | 參與者 |
|------|------|------|--------|
| Phase 1：模型審核 | 第 1 週 | 知識樹骨架 + 分層邏輯 | 全體獸醫師 |
| Phase 2：內容試讀 | 第 2 週 | AI 生成教學內容（3 個範例節點） | 全體獸醫師 |
| Phase 3：互動測試 | 第 3 週 | 測驗 + 病例闖關 + 複習流程 | 全體獸醫師 |
| Phase 4：總結報告 | 第 4 週 | 彙整反饋 + 改善建議 | 核心成員 |

## 三、Phase 1：知識模型審核

### 3.1 審核範圍

以**心臟科（CARD）**為範例專科，審核以下架構：

```
Layer 0: 學科總覽（心臟科概論）
Layer 1: 基礎科學（心臟解剖、心臟電生理、血流動力學...）
Layer 2: 病理機轉（瓣膜退行性變化、心肌病變機轉...）
Layer 3: 臨床疾病（MMVD、DCM、心絲蟲、心包膜積液...）
Layer 4: 診斷方法（心臟超音波、ECG、胸腔X光、生物標記...）
Layer 5: 治療管理（Pimobendan 用藥、利尿劑治療、飲食管理...）
```

### 3.2 評估標準

請針對每個面向給予 1-5 分，並附書面說明：

| 評估面向 | 說明 | 評分 |
|---------|------|------|
| **分層合理性** | 六層結構是否反映臨床學習的邏輯順序？ | 1-5 |
| **完整性** | 是否遺漏重要疾病、診斷或治療方法？ | 1-5 |
| **台灣在地化** | 是否包含台灣高盛行率疾病（如心絲蟲）？ | 1-5 |
| **難度分佈** | 從入門到進階是否涵蓋合理的難度梯度？ | 1-5 |
| **節點粒度** | 每個節點的範圍是否適中（不過大也不過小）？ | 1-5 |
| **關聯合理性** | 節點間的前置依賴、跨科關聯是否正確？ | 1-5 |

### 3.3 關聯類型審核

系統使用八種節點關聯類型，請確認是否涵蓋所有必要的知識連結：

| 關聯類型 | 說明 | 範例 |
|---------|------|------|
| prerequisite | 前置依賴 | 心臟解剖 → 心臟超音波 |
| builds_on | 延伸進階 | 心臟聽診基礎 → 心雜音分級 |
| cross_specialty | 跨科相關 | 腎臟病 ↔ 高血壓性心臟病 |
| differential | 鑑別比較 | DCM vs. MMVD |
| clinical_apply | 臨床應用 | 病理機轉 → 治療策略 |
| complication | 併發關聯 | MMVD → 肺水腫 |
| same_system | 同系統 | 心房 → 心室 |
| drug_related | 藥物相關 | Pimobendan → Furosemide |

## 四、Phase 2：內容品質評估

### 4.1 評估內容

AI 會為以下 3 個範例節點生成完整教學內容：

1. **CARD_010**：二尖瓣退行性病變 (MMVD) — Layer 3 疾病型
2. **CARD_020**：心臟超音波基礎 — Layer 4 診斷型
3. **CARD_001**：心臟科總覽 — Layer 0 概念型

### 4.2 評估標準

| 評估面向 | 說明 | 評分 |
|---------|------|------|
| **醫學正確性** | 內容是否有事實錯誤？藥物劑量是否準確？ | 1-5 |
| **深度適當性** | 對目標難度等級，深度是否恰當？ | 1-5 |
| **臨床實用性** | 內容是否有助於臨床決策？ | 1-5 |
| **學習結構** | 摘要→學習目標→主體→測驗的流程是否流暢？ | 1-5 |
| **術語準確性** | 中英文術語使用是否正確？ | 1-5 |
| **參考文獻** | 引用的教科書/指引是否為最新且權威？ | 1-5 |

### 4.3 疾病型特殊檢查（MMVD 範例）

- [ ] signalment（好發品種/年齡）是否正確
- [ ] ACVIM 分期是否使用最新版
- [ ] 鑑別診斷清單是否完整
- [ ] 治療方案是否符合台灣現有藥物
- [ ] 飼主溝通建議是否實用

## 五、Phase 3：互動功能測試

### 5.1 測驗引擎

請完成至少 1 組完整測驗（13 題），評估：

- [ ] 題目是否測到核心知識點
- [ ] 選項干擾項是否為常見迷思（不是荒謬選項）
- [ ] 解答說明是否清楚
- [ ] 難度是否匹配節點等級

### 5.2 病例闖關（7 步驟）

請完成至少 1 個完整病例闖關：

| 步驟 | 評估重點 |
|------|---------|
| 1. 病史收集 | 可選的問題是否涵蓋關鍵病史？ |
| 2. 理學檢查 | 檢查選項是否合理？發現是否正確？ |
| 3. 鑑別診斷 | 鑑別清單是否合理？ |
| 4. 診斷計畫 | 檢查選項是否有成本效益考量？ |
| 5. 結果判讀 | 結果描述是否擬真？ |
| 6. 治療計畫 | 治療方案是否符合指引？ |
| 7. 病例總結 | 回顧是否有教育價值？ |

### 5.3 間隔重複

- [ ] 複習排程是否合理（SM-2 演算法）
- [ ] 題目重複出現的頻率是否適當
- [ ] 熟練度追蹤是否反映真實記憶狀態
